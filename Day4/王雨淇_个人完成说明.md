## 完成结果

### 1. kb_document 数据库表
- [x] 15 个字段与 Day4 文档约定完全一致：id / title / original_name / stored_name / file_path / file_type / file_size / chunk_count / status / process_stage / error_message / create_user_id / create_time / update_time / deleted
- [x] 状态字段支持 PENDING / PROCESSING / COMPLETED / FAILED 四种状态
- [x] 处理阶段字段支持 UPLOADED / EXTRACTING / SPLITTING / EMBEDDING / INDEXING / DONE 六阶段
- [x] 5 个性能索引：status / file_type / create_user_id / create_time / deleted
- [x] utf8mb4_unicode_ci 字符集，InnoDB 引擎
- [x] 在 Docker MySQL 8.0 容器中成功建表并验证（DESC + SHOW INDEX）

### 2. KbDocument 实体与 Mapper
- [x] KbDocument.java 遵循 User.java 代码风格（手动 getter/setter，非 Lombok @Data）
- [x] @TableName("kb_document") 映射表名
- [x] @TableId(type = IdType.AUTO) 主键自增
- [x] @TableLogic 逻辑删除（deleted 字段）
- [x] @TableField(fill = FieldFill.INSERT / INSERT_UPDATE) 自动填充 createTime / updateTime
- [x] KbDocumentMapper.java 继承 BaseMapper\<KbDocument\>，自动获得 CRUD + 分页 + 逻辑删除能力

### 3. DTO / VO 层
- [x] DocumentQueryDTO：page 默认 1、size 默认 10，支持 keyword 搜索和 status 筛选
- [x] DocumentUpdateDTO：title 可选，不传则保持原标题
- [x] DocumentVO：@JsonInclude(NON_NULL)，不暴露 filePath 字段，仅返回 fileSize 数值

### 4. FileStorageUtil 文件工具
- [x] UUID 随机命名存储文件，保留原扩展名
- [x] 格式校验：仅允许 .pdf / .doc / .docx / .txt
- [x] 大小校验：最大 50MB（与前端 beforeUpload 一致）
- [x] 路径穿越防护：save() 和 deleteFile() 均校验路径在 uploadDir 内
- [x] save(MultipartFile)：保存文件并返回 StoredFile（storedName + absolutePath）
- [x] replace(oldPath, newFile)：先存新文件 → 成功后再删旧文件（防止新文件失败导致旧文件丢失）
- [x] deleteFile(path)：幂等删除，路径不在 uploadDir 内时抛 BizException

### 5. AsyncConfig 异步线程池
- [x] @EnableAsync 开启异步支持
- [x] docProcessExecutor Bean：corePoolSize=2, maxPoolSize=4, queueCapacity=50
- [x] CallerRunsPolicy 拒绝策略（队列满时不丢弃任务）

## 组长审查发现的 Bug 及修复

### Bug #1（replace 先删后存）
- 问题：replace() 先 deleteFile(oldFilePath) → save(newFile)，新文件保存失败时旧文件已丢失
- 修复：改为先 save(newFile) → 保存成功后再 deleteFile(oldFilePath)；新旧路径相同时跳过删除

### Bug #2（deleteFile 无路径边界检查）
- 问题：deleteFile() 未校验目标路径在 uploadDir 内，可能误删系统文件
- 修复：删除前 Paths.get(absolutePath).normalize().startsWith(root)，不满足时抛 BizException("非法文件路径")
