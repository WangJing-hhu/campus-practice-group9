## 遇到问题

1. **本机无法通过 git clone 获取仓库代码**：GitHub HTTPS 443 端口连接被重置，git clone 和 git push 均失败
2. **本机未安装 MySQL**：没有 MySQL 命令行客户端，无法执行建表 SQL
3. **WebFlux 与 Spring MVC 共存问题**：Day4 需要 WebClient 调用 Python AI，但不能影响已有的 Spring MVC 架构
4. **FileStorageUtil.replace() 先删后存的数据丢失风险**：组长审查指出如果新文件保存失败，旧文件已被删除
5. **FileStorageUtil.deleteFile() 缺少路径边界检查**：组长审查指出没有校验目标路径在 uploadDir 内，可能误删系统文件
6. **mvn test 失败**：Day2IntegrationTest（集成测试）依赖本地 MySQL 连接，MySQL 容器端口与配置不一致导致测试报错

## 解决过程

**问题 1（GitHub 网络）**：
- 通过 `gh api` 批量下载仓库中 47 个源文件（Python 脚本 + base64 解码），在本地 `C:\Users\Miss.W\campus-qa-test\` 组装出完整项目
- 通过 GitHub Contents API（PUT 请求）逐文件提交代码，绕过 git 协议限制
- 大文件（PDF/DOCX）通过 JSON 临时文件 + `--input` 参数提交

**问题 2（MySQL 安装）**：
- 使用 Docker 启动 MySQL 8.0 容器：`docker run -d --name campus-mysql-day4 -e MYSQL_ROOT_PASSWORD=root123 -e MYSQL_DATABASE=campus_qa -p 3307:3306 mysql:8.0`
- 通过 Docker 运行 mysql 客户端连接容器：`docker run --rm mysql:8.0 mysql -h host.docker.internal -P 3307 -u root -proot123`
- 成功执行 `sql/day4_kb_document.sql` 建表，DESC 和 SHOW INDEX 验证通过

**问题 3（WebFlux 与 MVC 共存）**：
- 查阅 Spring Boot 3.2 官方文档确认：同时引入 spring-boot-starter-web 和 spring-boot-starter-webflux 时，Spring Boot 自动配置为 MVC 模式（WebFlux 不接管 Web 层）
- WebClient 仅作为 HTTP 客户端使用，不影响 Tomcat 和 DispatcherServlet
- pom.xml 中只在注释中标注"仅使用 WebClient 调用 Python AI"

**问题 4（replace 先删后存）**：
- 原代码：`deleteFile(oldPath); return save(newFile);`
- 修复后：`StoredFile f = save(newFile); if(!oldPath.equals(f.getPath())) deleteFile(oldPath); return f;`
- 新文件保存失败时（格式/大小/磁盘满），旧文件完全不受影响

**问题 5（deleteFile 缺路径检查）**：
- 修复前：直接 `Files.deleteIfExists(Paths.get(absolutePath))`，无任何路径校验
- 修复后：`Path root = Paths.get(uploadDir).toAbsolutePath().normalize(); Path target = Paths.get(absolutePath).toAbsolutePath().normalize(); if(!target.startsWith(root)) throw BizException("非法文件路径");`
- 与 save() 方法中的路径穿越防护保持一致

**问题 6（mvn test 失败）**：
- ApplicationSmokeTest 使用 `@SpringBootTest` 无数据库依赖，8/8 全部通过
- Day2IntegrationTest 使用 `@ActiveProfiles("dev")` 需要 MySQL，环境变量 DB_PORT=3307 后重试，但仍需完整用户表和数据初始化
- 结论：Day2IntegrationTest 是 Day2 基线遗留问题，与 Day4 新增代码无关，联调阶段统一验证
