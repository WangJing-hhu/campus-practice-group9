# Day4 接口、状态与元数据说明

## 1. 文档目的

本文档用于统一校园问答助手 Day4 知识库模块的接口路径、权限规则、处理状态和元数据字段，作为 React 前端、Spring Boot 后端与 Python FastAPI AI 服务联调时的共同依据。

系统调用关系如下：

```text
React 前端
    ↓
Spring Boot 后端
    ├─ MySQL：保存文档元数据与处理状态
    ├─ storage/uploads：保存原始文件
    └─ AiClient / WebClient
            ↓
       Python FastAPI AI 服务
            ├─ 文本读取
            ├─ 文本切分
            ├─ Embedding
            └─ FAISS 向量索引
```

前端只调用 Spring Boot，不直接调用 Python AI 服务。

## 2. 服务地址与数据格式

Spring Boot 默认地址：

```text
http://localhost:8081
```

Python FastAPI 默认地址：

```text
http://localhost:8000
```

普通业务接口使用：

```http
Content-Type: application/json
```

文件上传接口使用：

```http
Content-Type: multipart/form-data
```

预览和下载接口返回文件流。

普通 JSON 接口采用统一返回结构：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

分页结果示例：

```json
{
  "records": [],
  "total": 0,
  "current": 1,
  "size": 10
}
```

## 4. Spring Boot 文档管理接口

### 4.1 上传文档

```http
POST /api/doc/upload
```

权限：仅管理员。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `file` | File | 是 | 上传文件 |
| `title` | String | 否 | 文档标题 |

支持格式：

```text
PDF、DOC、DOCX、TXT
```

最大文件大小：

```text
50 MB
```

处理流程：

```text
格式与大小校验
    ↓
生成 UUID 存储名
    ↓
保存到 storage/uploads
    ↓
写入 kb_document
    ↓
初始化为 PENDING / UPLOADED
    ↓
异步调用 Python /process
```

### 4.2 查询文档列表

```http
GET /api/doc/list
```

权限：仅管理员。

可选参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| `keyword` | String | 按标题或原文件名搜索 |
| `status` | String | 按状态筛选 |
| `current` | Integer | 当前页 |
| `size` | Integer | 每页数量 |

示例：

```http
GET /api/doc/list?keyword=校园&status=COMPLETED&current=1&size=10
```

### 4.3 查询文档详情

```http
GET /api/doc/{id}
```

权限：仅管理员。

主要返回标题、原文件名、文件类型、文件大小、处理状态、处理阶段、切分块数、错误信息和创建时间。

### 4.4 预览文档

```http
GET /api/doc/{id}/preview
```

权限：公开访问，不要求 JWT。

| 文件类型 | 处理方式 |
|---|---|
| PDF | 浏览器内嵌预览 |
| TXT | 浏览器文本预览 |
| DOC | 提示下载 |
| DOCX | 提示下载 |

PDF 推荐响应头：

```http
Content-Type: application/pdf
Content-Disposition: inline
```

TXT 推荐响应头：

```http
Content-Type: text/plain;charset=UTF-8
Content-Disposition: inline
```

安全要求：

1. 前端只传文档 ID，不传服务器路径。
2. 后端根据文档 ID 查询真实文件位置。
3. 文件必须位于 `storage/uploads` 目录。
4. 不得在错误信息中暴露绝对路径。
5. 文档或物理文件不存在时返回 404。

### 4.5 下载文档

```http
GET /api/doc/{id}/download
```

权限：公开访问，不要求 JWT。

推荐响应头：

```http
Content-Disposition: attachment
```

下载名称使用 `originalName`，不得直接使用服务器内部 UUID 文件名。

### 4.6 更新文档

```http
PUT /api/doc/{id}
```

权限：仅管理员。

请求示例：

```json
{
  "title": "河海大学校园生活指南（修订版）"
}
```

### 4.7 重新处理文档

建议接口：

```http
POST /api/doc/{id}/reprocess
```

权限：仅管理员。

处理流程：

```text
校验文档和原始文件
    ↓
删除旧向量和旧元数据
    ↓
状态重置为 PENDING / UPLOADED
    ↓
异步调用 Python /process
    ↓
重新读取、切分、向量化和入库
```

最终路径以 Java 后端实际实现为准。

### 4.8 删除文档

```http
DELETE /api/doc/{id}
```

权限：仅管理员。

删除流程：

```text
查询文档记录
    ↓
调用 Python 删除向量
    ↓
删除或逻辑删除数据库记录
    ↓
删除原始文件
    ↓
返回结果
```

### 4.9 知识库检索

```http
POST /api/doc/search
```

权限：仅管理员。

请求示例：

```json
{
  "question": "河海大学图书馆几点开放？",
  "topK": 5
}
```

结果主要包含：

```text
content
docId
chunkIndex
title
score
```

## 5. Python FastAPI 内部接口

Python AI 服务仅供 Java 后端调用。

### 5.1 健康检查

```http
GET /health
```

返回示例：

```json
{
  "status": "OK",
  "model": "text-embedding-v3",
  "index_size": 120
}
```

### 5.2 处理文档

```http
POST /process
```

请求示例：

```json
{
  "doc_id": 1,
  "path": "storage/uploads/uuid-file.pdf",
  "title": "河海大学校园生活指南",
  "callback_url": "http://localhost:8081/api/doc/callback"
}
```

处理流程：

```text
删除旧向量
    ↓
读取原始文件
    ↓
文本切分
    ↓
生成 Embedding
    ↓
写入 FAISS
    ↓
保存 metadata.json
    ↓
更新处理状态
```

返回示例：

```json
{
  "doc_id": 1,
  "chunk_count": 18,
  "status": "COMPLETED"
}
```

### 5.3 向量检索

```http
POST /search
```

请求示例：

```json
{
  "question": "河海大学图书馆几点开放？",
  "top_k": 5
}
```

### 5.4 删除文档向量

```http
DELETE /document/{doc_id}
```

返回示例：

```json
{
  "doc_id": 1,
  "deleted_vectors": 18,
  "status": "OK"
}
```

删除接口应具备幂等性。

## 6. 文档处理状态

### 6.1 总体状态 `status`

| 状态 | 含义 |
|---|---|
| `PENDING` | 文件已上传，等待处理 |
| `PROCESSING` | 正在处理 |
| `COMPLETED` | 已成功写入向量库 |
| `FAILED` | 某一阶段处理失败 |

正常流转：

```text
PENDING
   ↓
PROCESSING
   ↓
COMPLETED
```

异常流转：

```text
PENDING
   ↓
PROCESSING
   ↓
FAILED
```

### 6.2 处理阶段 `processStage`

| 阶段 | 含义 |
|---|---|
| `UPLOADED` | 原始文件已保存 |
| `EXTRACTING` | 正在提取文本 |
| `SPLITTING` | 正在切分文本 |
| `EMBEDDING` | 正在生成向量 |
| `INDEXING` | 正在写入 FAISS |
| `DONE` | 全部处理完成 |

完整流程：

```text
PENDING / UPLOADED
        ↓
PROCESSING / EXTRACTING
        ↓
PROCESSING / SPLITTING
        ↓
PROCESSING / EMBEDDING
        ↓
PROCESSING / INDEXING
        ↓
COMPLETED / DONE
```

发生异常时：

```text
status = FAILED
errorMessage = 具体错误信息
```

## 7. MySQL 文档元数据

`kb_document` 表核心字段：

| 字段 | 说明 |
|---|---|
| `id` | 文档唯一编号 |
| `title` | 文档标题 |
| `original_name` | 原始文件名 |
| `stored_name` | 服务器存储文件名 |
| `file_path` | 服务器内部文件路径 |
| `file_type` | 文件类型 |
| `file_size` | 文件大小 |
| `status` | 总体处理状态 |
| `process_stage` | 当前处理阶段 |
| `chunk_count` | 文本块数量 |
| `error_message` | 失败信息 |
| `created_by` | 上传用户 ID |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |
| `deleted` | 逻辑删除标记 |

`file_path`、绝对路径、数据库连接信息、API Key 和内部 Token 不得返回前端。

## 8. FAISS 向量元数据

Python AI 服务通过以下文件持久化数据：

```text
faiss.index
metadata.json
```

每个文本块至少包含：

| 字段 | 说明 |
|---|---|
| `vector_id` | 向量唯一编号 |
| `doc_id` | 所属文档 ID |
| `chunk_idx` | 文本块顺序 |
| `content` | 文本块内容 |
| `title` | 来源文档标题 |

示例：

```json
{
  "vector_id": 12,
  "doc_id": 1,
  "chunk_idx": 3,
  "content": "图书馆开放时间为每天 8:00 至 22:00……",
  "title": "河海大学校园生活指南"
}
```

重新处理前，应先按 `doc_id` 删除已有向量和元数据。

## 9. 文件存储约定

原始文件统一保存到：

```text
storage/uploads
```

保存时使用 UUID 等唯一名称，避免同名覆盖。

用户下载时使用 `originalName`，服务器内部读取时使用 `storedName` 和 `filePath`。

Java 后端与 Python 服务的 `UPLOAD_DIR` 必须指向同一个实际目录。

---

## 10. 错误处理

| HTTP 状态码 | 含义 |
|---|---|
| `200` | 请求成功 |
| `400` | 参数错误或文件不符合要求 |
| `401` | 未登录或内部 Token 无效 |
| `403` | 权限不足 |
| `404` | 文档或物理文件不存在 |
| `409` | 当前状态不允许执行操作 |
| `500` | 服务器内部处理失败 |
| `503` | Python AI 服务不可用 |

错误信息不得包含数据库密码、API Key、服务器绝对路径或内部 Token。

## 11. 字段命名约定

Java 与前端使用驼峰命名：

```text
originalName
fileType
fileSize
processStage
chunkCount
errorMessage
createdAt
updatedAt
```

Python 内部接口使用下划线命名：

```text
doc_id
chunk_idx
chunk_count
callback_url
top_k
```

字段转换示例：

```text
docId      → doc_id
chunkCount → chunk_count
topK       → top_k
```

## 12. 联调检查重点

1. 管理员能否上传 PDF、DOC、DOCX 和 TXT。
2. 超过 50MB 或格式不支持的文件能否被拦截。
3. 上传后是否生成 `kb_document` 记录。
4. 状态是否从 `PENDING` 进入 `PROCESSING`。
5. Python 能否访问 Java 保存的实际文件。
6. 文档是否完成读取、切分、Embedding 和 FAISS 入库。
7. `chunkCount` 是否正确回写。
8. `COMPLETED` 文档能否被检索。
9. 重新处理是否先删除旧向量。
10. 删除后数据库、文件和向量是否同步清理。
11. 未登录用户能否正常预览和下载。
12. 普通用户访问管理员接口是否返回 403。
13. 前端是否隐藏 `filePath` 等内部字段。
14. Python 服务未启动时是否返回明确错误。
15. 失败后是否记录 `errorMessage`。

## 13. 安全约定

1. DashScope API Key 不提交到 GitHub。
2. MySQL 密码不提交到 GitHub。
3. 内部 Token 不写入公开代码。
4. 使用 `.env` 或环境变量保存敏感配置。
5. `.env` 必须加入 `.gitignore`。
6. 文件路径必须经过安全目录校验。
7. 禁止按前端传入的任意路径读取文件。
8. 预览和下载接口不得暴露绝对路径。
9. 删除接口必须根据数据库记录定位文件。
10. Python 回调地址应限制为可信后端地址。
